Parche para compilar Deauther con esp_wifi_80211_tx
====================================================

El archivo src/Deauther.cpp redefine:

extern "C" int ieee80211_raw_frame_sanity_check(...)

Para que el linker acepte ese override, primero hay que debilitar el simbolo
ieee80211_raw_frame_sanity_check dentro de libnet80211.a.

ESP32-S3 / CYBERDECK-MINI-ESP32:

C:\Users\PepeAngell\.platformio\packages\toolchain-xtensa-esp32s3\bin\xtensa-esp32s3-elf-objcopy.exe --weaken-symbol=ieee80211_raw_frame_sanity_check C:\Users\PepeAngell\.platformio\packages\framework-arduinoespressif32\tools\sdk\esp32s3\lib\libnet80211.a C:\Users\PepeAngell\.platformio\packages\framework-arduinoespressif32\tools\sdk\esp32s3\lib\libnet80211.a

ESP32 clasico:

C:\Users\PepeAngell\.platformio\packages\toolchain-xtensa-esp32\bin\xtensa-esp32-elf-objcopy.exe --weaken-symbol=ieee80211_raw_frame_sanity_check C:\Users\PepeAngell\.platformio\packages\framework-arduinoespressif32\tools\sdk\esp32\lib\libnet80211.a C:\Users\PepeAngell\.platformio\packages\framework-arduinoespressif32\tools\sdk\esp32\lib\libnet80211.a

Si PlatformIO reinstala el framework, borra .platformio/packages o cambia de
familia de chip, hay que aplicar otra vez el comando que corresponda.
